# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in the skills, the landing page, or any related infrastructure, please email **security@cleolabs.co**.

**Do not open public issues for security concerns.**

## Response Timeline

- Initial response: within 48 hours
- Triage and assessment: within 5 business days
- Fix or mitigation: depends on severity, typically 30 days for critical issues

## Supported Versions

Only the latest version (`main` branch) is supported. We do not backport security fixes to older releases.

| Version | Supported |
|---------|-----------|
| main    | ✅ |
| < main  | ❌ |

## Disclosure Policy

We follow coordinated disclosure. After fixing a vulnerability, we will:
1. Publish a security advisory on GitHub
2. Credit the reporter (if desired)
3. Update CHANGELOG.md
